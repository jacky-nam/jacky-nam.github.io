01 Work in progress

Currently working on my capstone: a complete ultrasound platform. Five custom boards — main control, transducer driver, analog front-end, display/image-processing, and power management — plus a PyTorch U-Net that denoises the reconstructed image.

The hard part isn't any one piece; it's that a returning echo is measured in microvolts and everything between the transducer and the screen is a chance to ruin it.

  • ● Hardware — five-board signal chain, low-noise analog routing, EMI separation
  • ● ML — U-Net denoiser, residual-noise target, verified on GPU before real data
  • ◦ Next — Field II simulated data, real ultrasound metrics, MPS inference

// How I work

A few things I believe about building:

  • Verify the trivial version first. A training loop has to overfit four images before it sees real data; a noise budget is set before a trace is routed.
  • Fail loudly, early. If CUDA isn't live, the script exits — it doesn't limp along on CPU and waste an afternoon.
  • Swap one thing at a time. The denoiser was built so real data drops in by changing only the Dataset class. Nothing else moves.
  • The boring layers matter most. Power and grounding decide whether the clever parts get to work at all.
